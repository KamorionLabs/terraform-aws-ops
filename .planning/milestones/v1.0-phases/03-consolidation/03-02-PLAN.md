**run_mysqldump_on_eks consolidation (hybrid pattern -- divergence inside Map):**

1. Keep shared outer flow states unchanged: EnsureClusterAvailable, GetClusterInfo, EnsureNodegroupCapacity, GetDbSecretValue, FormatSecretString, DumpTablesMap (Map state), PrepareOutput, DumpFailed, DumpSucceeded.

2. Insert `CheckAccessMode` Choice state between EnsureNodegroupCapacity and the next state:
   - `$.EKS.AccessMode` StringEquals `"public"` -> `GetEksClusterInfo`
   - Default -> `GetDbSecretValue` (skip GetEksClusterInfo for private)

3. Add `GetEksClusterInfo` state from the public variant (eks:describeCluster, ResultPath $.EksCluster). Its Next -> GetDbSecretValue.

4. Update DumpTablesMap ItemSelector to include BOTH sets of fields:
   - Public fields: `ClusterName.$: $.EksCluster.Cluster.Name`, `CertificateAuthority.$: $.EksCluster.Cluster.CertificateAuthority.Data`, `Endpoint.$: $.EksCluster.Cluster.Endpoint`
   - Private fields: `K8sProxyLambdaArn.$: $.K8sProxyLambdaArn`, `EksClusterName.$: $.EksClusterName`
   - Routing field: `AccessMode.$: $.EKS.AccessMode`
   - Keep existing shared fields from both variants (DestinationAccount, Namespace, DbSecret, table iterator fields)
   - IMPORTANT: Public $.EksCluster may not exist when AccessMode=private. Use conditional ItemSelector fields if possible, or accept that undefined paths will result in null values (SFN tolerates this if the value is never used in the chosen branch).

5. Inside Map ItemProcessor:
   - Add `CheckAccessModeForDump` Choice state as StartAt:
     - `$.AccessMode` StringEquals `"public"` -> `RunEksJobForTable` (from public variant, uses eks:runJob.sync)
     - Default -> `CreateDumpJob` (from private variant, uses lambda:invoke)
   - Include the public single-state path: RunEksJobForTable -> End
   - Include the private multi-state cycle: CreateDumpJob -> WaitForDumpJob -> CheckDumpJobStatus -> DumpJobStatusChoice -> (loop or) DeleteDumpJob -> End, plus DeleteDumpJobOnError -> DumpJobFailed
   - Keep all Retry, Catch, ResultSelector blocks from their original variant

**run_mysqlimport_on_eks consolidation (complex pattern -- divergence at outer flow):**

1. Keep shared states: EnsureClusterAvailable, GetClusterInfo, EnsureNodegroupCapacity, GetDbSecretValue, FormatSecretString, PrepareOutput, ImportFailed, ImportSucceeded.

2. Insert `CheckAccessModeForImport` Choice state after FormatSecretString (the last shared prep state):
   - `$.EKS.AccessMode` StringEquals `"public"` -> `GetEksClusterInfo`
   - Default -> `CreateImportJob`

3. Public branch (2 states from public variant):
   - `GetEksClusterInfo` (eks:describeCluster, ResultPath $.EksCluster) -> `RunEksJobForImport` (eks:runJob.sync) -> PrepareOutput
   - Keep all Parameters, Retry, Catch from the public original

4. Private branch (8 states from private variant):
   - `CreateImportJob` (lambda:invoke) -> `WaitForImportJob` (Wait) -> `CheckImportJobStatus` (lambda:invoke) -> `ImportJobStatusChoice` (Choice)
   - ImportJobStatusChoice routes: Active/Running -> WaitForImportJob (loop), Complete/Succeeded -> `CheckSkipDeletion`, Failed -> `DeleteImportJobOnError`
   - `CheckSkipDeletion` (Choice): $.DebugKeepJob == true -> PrepareOutput, Default -> `DeleteImportJob`
   - `DeleteImportJob` (lambda:invoke) -> PrepareOutput
   - `DeleteImportJobOnError` (lambda:invoke) -> `ImportJobFailed` (Fail)
   - Keep all Parameters, Retry, Catch, ResultSelector from the private original

5. Also add a `CheckAccessMode` Choice state at the beginning (after EnsureNodegroupCapacity, before GetDbSecretValue) for the GetEksClusterInfo skip pattern, same as run_mysqldump. OR: if the public variant has GetEksClusterInfo between EnsureNodegroupCapacity and GetDbSecretValue, insert the Choice there. If GetEksClusterInfo is after FormatSecretString in the public variant, insert the choice there instead. Read the actual file to determine the correct insertion point.

After creating each consolidated file, delete the corresponding _private file.
