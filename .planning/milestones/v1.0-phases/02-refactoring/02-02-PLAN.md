**Step 1: Create ensure_snapshot_available.asl.json** in `modules/step-functions/db/`.
- Follow Phase 1 sub-SFN patterns: Comment with I/O contract, named Fail state (EnsureSnapshotAvailableFailed), self-contained Catch on all Task states.
- Input: {SnapshotIdentifier, Account: {RoleArn}, ProxyLambdaArn?, SourceRegion?}
- Output: {SnapshotIdentifier, SnapshotArn, Status: "available", EngineVersion}
- States (~7-8):
  1. CheckProxyForDescribe (Choice): routes to proxy or direct based on ProxyLambdaArn presence
  2. DescribeSnapshotDirect (Task): `aws-sdk:rds:describeDBClusterSnapshots` with Credentials
  3. DescribeSnapshotViaProxy (Task): `lambda:invoke` for cross-region proxy
  4. NormalizeDescribeResult (Pass): unify output format from both paths
  5. IsAvailable (Choice): check Status == "available"
  6. WaitSnapshot (Wait): 30 seconds
  7. SnapshotAvailable (Pass): format output -> Succeed
  8. EnsureSnapshotAvailableFailed (Fail): named error
- The Wait/Describe/Check loop continues until status is "available".
- No snapshot creation -- only wait/verify (per locked decision: "Pas de creation, geree en amont").
- This is a file() sub-SFN (no templatefile variables needed -- it uses direct SDK calls).
