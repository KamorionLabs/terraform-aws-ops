Cross-account client pattern (from fetch_secrets.py lines 38-76):
```python
def get_cross_account_client(service, role_arn, region, session_name="SyncConfigItems"):
    sts_client = boto3.client("sts")
    assumed_role = sts_client.assume_role(
        RoleArn=role_arn, RoleSessionName=session_name, DurationSeconds=900
    )
    credentials = assumed_role["Credentials"]
    return boto3.client(service, region_name=region,
        aws_access_key_id=credentials["AccessKeyId"],
        aws_secret_access_key=credentials["SecretAccessKey"],
        aws_session_token=credentials["SessionToken"])
```

Merge mode rules (from CONTEXT.md locked decisions):
- MergeMode=true: destination wins for common keys without Transform; source only overwrites keys with explicit Transform; dest-only keys preserved
- MergeMode=false (default): total overwrite of destination by source (after Transforms); dest-only keys lost
- Non-JSON + MergeMode=true: if dest exists, keep dest value; if no dest, copy source (with Transforms)

Skip transform semantics:
- skip: true means "exclude this key from the synced value" (remove from result)
- In MergeMode, destination value is preserved for skipped keys

SM write pattern:
- Try put_secret_value first (update path, most common)
- Catch ResourceNotFoundException -> create_secret (create path)

SSM write pattern:
- put_parameter(Overwrite=True) with Type from source
- Type is required for new params, preserved for updates

SSM recursive:
- get_parameters_by_path(Recursive=True, WithDecryption=True)
- Path mapping applied per returned parameter

SM listing for wildcards:
- list_secrets via paginator, filter client-side with fnmatch (SM filter does NOT support glob)
- Use Filters=[{"Key": "name", "Values": [prefix]}] to narrow results where possible
