    Creer modules/step-functions/sync/sync_config_items.asl.json avec la structure suivante. Ce fichier utilise des placeholders templatefile() qui seront remplaces par le module Terraform (Plan 02).

    Structure ASL (conforme a la decision CONTEXT.md avec le Choice DANS le Map, per-item routing) :

    ```json
    {
      "Comment": "Sync config items (secrets/parameters) between AWS accounts - routes SM/SSM via Choice state per item",
      "StartAt": "ValidateInput",
      "States": {
        "ValidateInput": Choice state qui verifie :
          - $.ConfigSync.Enabled == true ET $.ConfigSync.Items IsPresent -> "MapOverItems"
          - Default -> "SyncSkipped"

        "MapOverItems": Map state avec :
          - ItemsPath: "$.ConfigSync.Items"
          - ItemSelector: { "Item.$": "$$.Map.Item.Value", "SourceAccount.$": "$.ConfigSync.SourceAccount", "DestinationAccount.$": "$.ConfigSync.DestinationAccount" }
          - MaxConcurrency: 1 (sequential pour eviter les race conditions sur les secrets)
          - ItemProcessor Mode INLINE avec StartAt "CheckType" :
            - "CheckType": Choice state :
              - $.Item.Type StringEquals "SecretsManager" -> "SyncSMItem"
              - $.Item.Type StringEquals "SSMParameter" -> "SyncSSMItem"
              - Default -> "UnsupportedType"
            - "SyncSMItem": Task state avec Resource "${SyncConfigItemsLambdaArn}", End: true, Retry sur Lambda.ServiceException + Lambda.TooManyRequestsException (IntervalSeconds 2, MaxAttempts 3, BackoffRate 2), Catch States.ALL -> "ItemFailed"
            - "SyncSSMItem": Task state identique a SyncSMItem (meme Lambda, meme Retry/Catch), End: true
            - "UnsupportedType": Fail state avec Error "UnsupportedTypeError", Cause "Item.Type must be SecretsManager or SSMParameter"
            - "ItemFailed": Fail state avec Error "SyncItemFailed", Cause "Failed to sync config item"
          - ResultPath: "$.SyncResults"
          - Next: "PrepareOutput"

        "PrepareOutput": Pass state avec Parameters { "Status": "Completed", "ModuleName": "sync_config_items", "ItemsProcessed.$": "$.SyncResults" }, Next: "SyncSucceeded"

        "SyncSkipped": Pass state avec Parameters { "Status": "Skipped", "ModuleName": "sync_config_items", "Reason": "ConfigSync disabled or no items" }, Next: "SyncSucceeded"

        "SyncSucceeded": Succeed state
      }
    }
    ```

    Points critiques :
    - Le placeholder `${SyncConfigItemsLambdaArn}` sera resolu par templatefile() dans le module Terraform (Plan 02)
    - Les deux Task states (SyncSMItem/SyncSSMItem) appellent la MEME Lambda mais sont separes pour permettre d'ajouter des parametres specifiques SM vs SSM en Phase 5 si necessaire
    - Retry pattern identique a audit_resource.asl.json (Lambda.ServiceException, Lambda.TooManyRequestsException)
    - Fail states dans le Map iterator avec Error nomme (pattern SUB-05 du projet)
    - MaxConcurrency: 1 pour la Phase 4 (sequential) -- pourra etre augmente en Phase 5 si necessaire

    Creer le repertoire modules/step-functions/sync/ s'il n'existe pas (mkdir -p).
