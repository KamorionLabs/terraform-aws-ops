For each of the 3 pairs, create a consolidated ASL file that merges both pub and priv variants. The pattern for all 3 simple pairs is:

**Consolidation pattern:**
1. Add `CheckAccessMode` as the new StartAt state — a Choice state checking `$.EKS.AccessMode`:
   - `StringEquals: "public"` -> `GetEksClusterInfo`
   - `Default` -> first private flow state (the original StartAt of the private variant)

2. Add `GetEksClusterInfo` state from the public variant (eks:describeCluster with ResultPath $.EksCluster). Its Next points to the original public flow.

3. For K8s interaction states that exist in both variants (they differ in Resource, Parameters, Retry, Catch, ResultSelector):
   - Rename the public variant to `<StateName>` (keep original name)
   - Add the private variant as `<StateName>Private`
   - Before each K8s interaction point, add a Choice state `CheckAccessModeFor<Operation>` that routes to the correct variant. Both variants converge at the same Next state.
   - Exception: if the flow forks early into two complete branches (public branch and private branch) that converge at output, that is also acceptable and simpler when there are many sequential K8s operations.

4. For states that are identical between pub and priv (Wait states, Pass states, Choice states, terminal states), keep a single copy.

5. For Map states: the ItemSelector must include BOTH public fields (ClusterName, CertificateAuthority, Endpoint from $.EksCluster.Cluster) AND private fields (K8sProxyLambdaArn, EksClusterName). The ItemProcessor contains both state variants with a CheckAccessMode Choice inside the Map routing to the correct variant. Use `$.AccessMode` within the Map ItemSelector (passed from the outer `$.EKS.AccessMode` so the Map iterator can access it).

**Critical details per pair:**

**manage_storage (18 pub / 17 priv):**
- Public-only: GetEksClusterInfo (1 state)
- 6 dual-variant K8s operations: DeletePvc, DeletePv, DeleteSc, CreateSc, CreatePv, CreatePvc
- Shared: ChooseAction, Wait states, GetEfsIdFromSsm, PrepareDeleteOutput, PrepareCreateOutput, ProcessFailed, ProcessSucceeded
- Use two-branch fork pattern: after CheckAccessMode, public branch goes GetEksClusterInfo -> ChooseAction with eks:call states, private branch goes ChooseActionPrivate with lambda:invoke states. Both branches converge at PrepareDeleteOutput/PrepareCreateOutput.

**scale_services (9 pub / 8 priv):**
- Public-only: GetEksClusterInfo (1 state)
- Dual-variant states inside Map: PatchService (public uses eks:call, private uses lambda:invoke)
- Dual-variant states outside Map: GetSecret, DeleteSecret (after CheckCleanupSecret)
- Shared: WaitAfterScaling, CheckCleanupSecret, PrepareOutput, ScaleFailed, ScaleSucceeded
- Map ItemSelector: include BOTH ClusterName/CertificateAuthority/Endpoint AND K8sProxyLambdaArn/EksClusterName plus AccessMode passed from outer scope
- Inside Map ItemProcessor: CheckAccessModeForPatch Choice -> PatchService (eks:call) or PatchServicePrivate (lambda:invoke), both converge at WaitForScale. Keep ServiceNotFoundSkip and WaitForScale shared.
- After CheckCleanupSecret: CheckAccessModeForSecret Choice -> GetSecret (eks:call) / GetSecretPrivate (lambda:invoke), then DeleteSecret / DeleteSecretPrivate

**verify_and_restart_services (6 pub / 5 priv):**
- Public-only: GetEksClusterInfo (1 state)
- Dual-variant states inside Map: GetService, RestartService
- Shared: GetExpectedSubPath, PrepareOutput, VerificationFailed, VerificationSucceeded + Map internals (ExtractCurrentSubPath, DecideIfRestartNeeded, SubPathCorrect, etc.)
- Map ItemSelector: include BOTH sets of fields + AccessMode
- Inside Map ItemProcessor: CheckAccessModeForGet Choice -> GetService / GetServicePrivate, then shared logic, then CheckAccessModeForRestart -> RestartService / RestartServicePrivate

After creating each consolidated file, delete the corresponding _private file.

Preserve all Comment, Retry, Catch, ResultSelector blocks exactly from their original source variant. Public eks:call states catch `EKS.404`, private lambda:invoke states catch `States.TaskFailed` with `Lambda.ServiceException` retries. Private lambda:invoke states have ResultSelector extracting `$.Payload.ResponseBody` and `$.Payload.StatusCode`.
