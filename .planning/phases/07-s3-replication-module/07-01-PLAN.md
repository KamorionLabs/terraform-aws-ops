<objective>
Author the two read-merge-write ASL state machines of the new `modules/step-functions/s3/` module: `setup_cross_account_replication` (REPL-01) and `delete_replication` (REPL-04). Both operate on the single S3 `ReplicationConfiguration` per source bucket via imperative assume-role, supporting hub-and-spoke fan-out (REPL-05) through a `MaxConcurrency:1` Map over `Destinations[]`.

Purpose: S3 stores ONE replication config per bucket and `PutBucketReplication` REPLACES it wholesale; managing a spoke independently requires read-merge-write keyed by a deterministic Rule ID. The source bucket is owned by an external stack, so every write assumes the source role at runtime and the module never mutates source data (validate-only versioning).
Output: 2 valid `*.asl.json` files mirroring the EFS setup/delete skeleton, NO Lambda, NO cross-region proxy.
